Particle System — Component Architecture Demo

What's happening

The particle simulation runs in Wasm (Rust compiled to wasm32). JS reads particle data via zero-copy pointer into linear memory. Canvas2D renders particles with alpha blending.

The wit/canvas.wit file defines the interface contract. Today we use wasm-bindgen as the bridge (the "polyfill"). Tomorrow, with the Component Model, any language implementing the draw-app world will Just Work — same interface, any language.

The Plan 9 parallel

Plan 9: process mounts /dev/draw → draws pixels. It doesn't know if /dev/draw is local GPU, remote display, or VNC.

Wasm: component imports wasmverse:canvas/surface → draws pixels. It doesn't know if the host is a browser, native window, or headless test.

Same pattern. Same power. Capability-based, namespace-scoped, transport-transparent.
